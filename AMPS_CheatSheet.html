⚡ AMPS — Frontend Cheat Sheet

JavaScript client · pub/sub · SOW · production · keep this at your desk

The one idea

Publishers send messages to a topic without knowing who listens. Subscribers ask a topic for what they want (optionally filtered). The AMPS engine routes it all. You (frontend) mostly subscribe and render.

Connection URI

// transport :// host : port /amps/ type
wss://amps-dev:9007/amps/json
  • wss = secure WebSocket (use in prod), ws = plain
  • type = json, fix, xml, bson…

Connect & log on

import { Client, Command } from 'amps'
const client = new Client('orders-ui-7f3a') // unique!
await client.connect('wss://host:9007/amps/json')
await client.logon('user', 'pass')

Publish

client.publish('MARKET_DATA',
  { Symbol:'AAPL', TradePrice:192.45 })

Subscribe (+ filter)

const onMsg = m => render(m.data)
await client.subscribe(onMsg, 'MARKET_DATA')

// with content filter (recommended: Command)
const cmd = new Command('subscribe')
  .topic('ORDERS')
  .filter("/Symbol = 'AAPL' AND /Price > 100")
await client.execute(cmd, onMsg)

Live dashboard (SOW)

// snapshot now + live updates forever
const cmd = new Command('sow_and_subscribe')
  .topic('orders')
  .filter("/status = 'OPEN'")
  .options('oof')
await client.execute(cmd, onMsg)

Message commands you handle

commandmeaning → action
group_beginsnapshot starting
sowexisting record → add row
plive publish → add/update row
oofleft the filter → remove row
group_endsnapshot complete
m.header.command()  m.header.sowKey()  m.data

Filter operators

  • = <> < > <= >= — compare
  • AND OR NOT ( ) — logic
  • LIKE — regex match (PCRE)
  • BEGINS WITH — prefix match
  • Fields use XPath: /Symbol, /Derived/TodayVWAP

Power features

  • Delta: delta_publish/delta_subscribe send only changed fields.
  • Replay: bookmark subscribe replays the transaction log from a point in time.
  • Queue: each message goes to exactly one consumer (vs pub/sub = copy to all).
  • HA: HAClient with a server list auto-fails-over and re-subscribes.

⚠️ Gotchas (read these)

  • Unique client name — AMPS uses it to detect duplicates.
  • Unsubscribe / disconnect on unmount — or you leak connections.
  • Throttle rendering — feeds can be 1000s msg/sec; batch DOM updates.
  • Filters run server-side — don't re-filter in the browser.
  • Handle OOF — or dashboards fill with stale rows.
  • Use wss + HAClient in prod — encrypted + survives failover.
  • Never swallow errors — a silent failure on a trading screen is dangerous.

Ask your team lead for

  • Dev connection URI (host/port/type)
  • Topic names + their message schemas
  • Which topics are SOW-enabled
  • The auth mechanism + your entitlements
