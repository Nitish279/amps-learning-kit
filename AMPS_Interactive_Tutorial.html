AMPS for JavaScript Developers

A hands-on intro to the Advanced Message Processing System — the real-time pub/sub engine you'll use to stream market data, orders, and trades into the browser. No prior experience needed. Scroll to the live blotter demo to see how it works.

The 7 concepts you must own

1 · Topic

A named stream of messages (e.g. ORDERS). Publishers send to it; subscribers receive from it.

2 · Message

The payload — usually JSON for frontend work. Belongs to one topic + one message type.

3 · Publish

Sending a message to a topic. A backend service does this; you mostly consume.

4 · Subscribe

Registering interest in a topic. AMPS then pushes new matching messages to you, live.

5 · Content filter

A SQL-WHERE-like clause run on the server: /Ticker='IBM' AND /Price>100.

6 · State of the World

A live "latest record per key" table AMPS maintains. Snapshot your UI instantly on load.

7 · Command

The low-level, fully-controllable API. subscribe()/sow() are shortcuts over it.

Code you'll write — annotated

Connect & publish

import { Client } from 'amps'

// transport :// host : port / amps / message-type
const uri = 'wss://127.0.0.1:9007/amps/json'
const client = new Client('examplePublisher') // unique name!

await client.connect(uri)
client.publish('messages', { hi: 'Hello, World!' })
client.disconnect()

Subscribe with a content filter

import { Client, Command } from 'amps'

const onMessage = message => render(message.data)

const cmd = new Command('subscribe')
  .topic('ORDERS')
  .filter("/Ticker = 'IBM' AND /Price > 100")

const subId = await client.execute(cmd, onMessage)

The dashboard pattern — sowAndSubscribe

const rows = new Map() // your in-memory view, keyed by sowKey

const onMessage = message => {
  const cmd = message.header.command()
  const key = message.header.sowKey()
  switch (cmd) {
    case 'sow':  // snapshot record
    case 'p':    // live insert/update
      rows.set(key, message.data); break
    case 'oof':  // out-of-focus: no longer matches filter
      rows.delete(key); break
  }
  renderTable(rows)
}

const command = new Command('sow_and_subscribe')
  .topic('orders').filter("/status = 'OPEN'").options('oof')

await client.execute(command, onMessage)
